# API契約: バーコード商品情報自動入力

**フィーチャー**: `014-barcode-auto-fill` | **日付**: 2026-05-16  
**バージョン**: v1

## 概要

バーコード読み取り/手入力で取得した JAN を使って既存 API `GET /api/products/{jan}` を呼び出し、アイテムフォームへ商品情報を反映するための frontend 連携契約を定義する。

本機能では backend endpoint の追加は行わず、既存 JAN 検索 API を再利用する。

## 利用エンドポイント

### `GET /api/products/{jan}`

- Path Parameter: `jan`（8桁または13桁数字）
- 認証: 既存 API ポリシーに従う

#### Response `200 OK`

```json
{
  "name": "商品名",
  "manufacturer": "メーカー名",
  "price": 1234
}
```

補足:
- `manufacturer` は未取得時 `null`
- `price` は未取得時 `null`

#### Error

| Status | Code | frontend の扱い |
|---|---|---|
| 400 | `VALIDATION_ERROR` | JAN形式エラーとして表示（再入力促進） |
| 404 | `PRODUCT_NOT_FOUND` | 商品未検出メッセージ表示 |
| 429 | `RATE_LIMITED` | 混雑メッセージ表示、手動再試行を案内 |
| 500 | `UPSTREAM_AUTH_FAILED` / `INTERNAL_SERVER_ERROR` | 取得失敗メッセージ表示 |
| 503 | `UPSTREAM_TIMEOUT` | 3秒以内失敗表示、手動再試行のみ |

## frontend 振る舞い契約

### 1. 検索起点

- カメラ読み取り成功時: 読み取り値をバーコード欄へ反映後、検索を実行する
- 手入力時: Enter 押下で検索を実行する

### 2. タイムアウト・再試行

- API 呼び出しは 3 秒でタイムアウト扱いにする
- 自動再試行は行わない
- 再試行はユーザー操作（ボタン/再実行）でのみ実施する

### 3. 競合時の値反映

- 既存入力値と取得値が競合する項目（商品名、メーカー、価格）は差分を表示する
- 項目ごとに既存値または取得値をユーザーが選択する

### 4. 欠損データ時の保存ルール

- 商品名欠損: 保存不可
- 価格またはメーカー欠損: 警告表示の上で保存可

### 5. レート制御

- 同時実行は 1 件まで
- 検索完了後 500ms は新規検索を受け付けない
- 連続入力時は前回検索をキャンセルし、最後の入力のみ処理する

## 非対象

- 新規 backend endpoint 追加
- JAN 以外のコード規格対応
- バーコード非対応ブラウザの動作保証
