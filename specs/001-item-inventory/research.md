# Research: 個人用物品管理

## 目的

この文書は、現在の仕様の実装に必要な技術選択と不明点を整理し、実装方針を明確にするための調査結果をまとめます。

## 技術選択の確認

### 1. 対象プラットフォーム

- **現状**: リポジトリに既存のアプリ実装が存在しないため、Web ブラウザ向けのシンプルな新規実装が最適。
- **選択**: Webアプリ。ブラウザで動作するシングルページアプリとして実装する。
- **理由**: 早期に動作確認でき、ログイン不要の個人用在庫管理として最小コストで成立する。

### 2. 永続化ストレージ

- **要件**: 今後の複数デバイス対応、クラウド展開、ログイン対応を見据えた永続化が必要。
- **選択**: バックエンドの SQL Server。
- **理由**: データをサーバー側で統合管理でき、複数デバイスからのアクセス、クラウドへの移行、ログイン/認証機能の追加に適している。
- **補足**: フロントエンド単体の `localStorage` / `IndexedDB` は現在の要件では適さないため、将来の拡張を見据えてバックエンド API + SQL Server 構成を採用する。

### 3. テスト戦略

- **要件**: ユーザー操作のテストとバリデーションの信頼性が必要。
- **候補**:
  - フロントエンド UI テスト: Playwright / Cypress
  - 単体テスト: Vitest / Vue Testing Library（フロントエンド）、xUnit（バックエンド）
  - データモデル/永続化テスト: Entity Framework Core のマイグレーションと SQL Server に対する統合テスト
- **推奨**: Vue と .NET 10 を前提に、フロントエンドは Vitest、バックエンドは xUnit を採用する。

## 仕様上の明確化

- 物品名称は在庫内で一意である。
- 数量は正の整数である必要がある。
- 現時点ではログインは不要だが、将来のログイン対応と複数デバイス同期を見据えた設計とする。
