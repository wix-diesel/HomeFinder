# Acceptance Checklist: アイテム詳細ページ操作（feature 006）

## US1: アイテム詳細を閲覧できる

- [X] 詳細ページが `/items/:id` でアクセスできる
- [X] アイテム名・数量・カテゴリ・保管場所が表示される
- [X] 存在しない ID にアクセスした場合に「見つかりません」メッセージを表示する
- [X] 取得失敗時にエラーメッセージを表示する
- [X] 「一覧に戻る」ボタンが表示される
- [X] 履歴ボタンは非活性で表示される

## US2: 編集ページへ遷移できる

- [X] 詳細ページのフッターに「編集」ボタンが表示される（`canEdit: true` 時）
- [X] 3点リーダーメニューに「編集」項目が表示される（`canEdit: true` 時）
- [X] 編集ボタン・メニュー押下で `/items/:id/edit` へ遷移する
- [X] `canEdit: false` の場合は編集ボタン・メニュー項目が表示されない

## US3: アイテムを削除できる

- [X] 3点リーダーメニューに「削除」が表示される（`canDelete: true` 時）
- [X] 削除前に確認ダイアログが表示される
- [X] 確認ダイアログでキャンセルした場合、削除が実行されない
- [X] 削除成功後にアイテム一覧ページへ遷移する
- [X] 削除後は一覧にそのアイテムが表示されない（論理削除）
- [X] 削除失敗（404）時は一覧ページへ遷移する
- [X] 削除失敗（その他）時はエラーメッセージを表示しページに留まる
- [X] `canDelete: false` の場合は削除メニュー項目が表示されない

## API・バックエンド

- [X] GET `/api/items/{id}` が `canEdit`, `canDelete` フィールドを含むレスポンスを返す
- [X] GET `/api/items/{id}` が存在しない ID に対して 404 を返す
- [X] DELETE `/api/items/{id}` が論理削除を実施して 204 を返す
- [X] DELETE `/api/items/{id}` が存在しない ID に対して 404 を返す
- [X] DELETE 後の GET が 404 を返す
- [X] DELETE 後の一覧 GET にそのアイテムが含まれない

## テスト

- [X] 契約テスト（contract）が全 PASS
- [X] 統合テスト（integration）が全 PASS
- [X] UI ユニットテスト（ItemDetailPage）が全 PASS
