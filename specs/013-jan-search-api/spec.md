# 機能仕様: JANコード検索 API

**フィーチャーブランチ**: `[013-jan-search-api]`
**作成日**: 2026-05-13
**ステータス**: ドラフト
**入力**: ユーザーによる説明: "JANバーコードをもとに商品を検索するAPIを用意する。楽天 RapidAPIで検索し、複数アイテムが見つかった場合は先頭のアイテムを返す。レスポンスに商品名・メーカー・価格を含め、見つからなければ404を返す。楽天 RapidAPIの設定は設定ファイルから取得する。UIはスコープ外。"

## 目的 (Why)

JANコードを与えることで、そのJANに該当する商品情報（商品名、メーカー、価格）を取得できる公開APIを提供する。商品データを外部の検索サービス（楽天 RapidAPI）から取得し、シンプルでテスト可能なレスポンスを返すことで、他のサービスやUIが商品情報を参照できるようにする。

## 要求 (Must / Should / Could)

Must

- FR-001: クライアントはJANコード（数字列）を入力してAPIを呼び出せること
- FR-002: システムは楽天 RapidAPIを用いてJANコードで商品検索を行うこと
- FR-003: 検索結果が1件以上ある場合、最初のアイテムをレスポンスとして返し、以下のフィールドを含むこと: 商品名、メーカー、価格
- FR-004: 該当商品が見つからない場合、HTTP 404 を返すこと
- FR-005: 楽天 RapidAPI に必要な認証情報（キー等）は外部設定ファイル（設定管理）から取得できること

Should

- SR-001: 価格が取得できない場合は価格フィールドを null または未設定で返すこと
- SR-002: 入力JANが不正（文字を含む、長さ不正など）の場合はHTTP 400 を返すこと

Could

- CR-001: 将来のために検索で取得した追加メタデータ（商品URL、画像URLなど）を拡張可能にすること

## ユーザーストーリー (優先度順)

### ユーザーストーリー 1 - JANで商品を取得する (優先度: P1)

エンドユーザー（または別のサービス）がJANコードを使って商品情報を取得したい。

**独立テスト**: JANコードを指定してAPIを呼び出し、期待するフィールドが返ることを確認する。

**受け入れシナリオ**:

1. **Given** 有効なJANコードが入力された場合, **When** APIを呼び出す, **Then** 商品名、メーカー、価格を含むレスポンスが返る。
2. **Given** 無効なJANコードが入力された場合, **When** APIを呼び出す, **Then** HTTP 400 エラーが返る。
3. **Given** JANコードに該当する商品が見つからない場合, **When** APIを呼び出す, **Then** HTTP 404 エラーが返る。

### ユーザーストーリー 2 - 見つからない場合の応答 (優先度: P1)

**受け入れシナリオ**:

1. Given 存在しないJAN, When APIにそのJANを問い合わせる, Then HTTP 404 を返す

## 受け入れ条件（AC）

- AC-001: 有効なJANを指定すると HTTP 200 と少なくとも商品名が返る
- AC-002: 検索結果が複数ある場合、先頭アイテムが返り、メーカーと価格フィールドが含まれている（価格は存在しない場合 null 可）
- AC-003: 存在しないJANでは HTTP 404 を返す
- AC-004: 不正な入力では HTTP 400 を返す
- AC-005: 楽天 RapidAPI の認証情報は設定ファイルから読み込まれることを確認できる

## エッジケース / 失敗時の挙動

- 無効なJANフォーマット（文字を含む、桁数不正） → HTTP 400
- 楽天 RapidAPI がタイムアウト/エラー → HTTP 502 または適切な 5xx を返し、エラーメッセージを含める
- レートリミットに達した場合 → 429 を返す（外部APIのエラーを透過するか、適切にハンドリングする）
- 検索結果に価格/メーカーが欠落している場合 → 欠落項目は null または空文字で返す（上位の調整で決定）
- 楽天 RapidAPI の認証情報が無効または期限切れの場合、HTTP 500 エラーを返す
- JANコードが空の場合、HTTP 400 エラーを返す
- 楽天 RapidAPI からのレスポンスがタイムアウトした場合、HTTP 503 エラーを返す

## 主要エンティティ

- JAN (文字列): 検索キー
- Product (レスポンスオブジェクト): { name, manufacturer, price }

## 成功基準 (Success Criteria)

- SC-001: 有効なJANで呼び出したとき、95% のリクエストが 2 秒以内に成功応答（200）を返す
- SC-002: 存在しないJANのリクエストは一貫して 404 を返す
- SC-003: 受け取ったレスポンスに必須フィールド（商品名、メーカー、価格）が含まれる割合が 99% 以上であること（価格欠落は null 許容）
- SC-004: API 定義と受け入れシナリオに従って手動テストで基本機能が検証できること

## 前提条件 / 依存

- 楽天 RapidAPI の利用契約と API キーが用意されていること
- 設定ファイルから認証情報が取得できる仕組みが存在すること
- この機能は API 層の提供に限定し、UI はスコープ外

## 用語

- **JANコード**: 日本国内で使用されるバーコード規格。13桁または8桁の数字列。
- **楽天 RapidAPI**: 楽天が提供する外部APIサービス。商品検索機能を含む。

## 仮定（Assumptions）

- 楽天 RapidAPI は商品検索に支持される信頼できるデータソースである
- JAN はユニークな検索キーとして機能する（ただし複数ヒットはあり得る）

## 明確化された仕様

### 機能範囲 (Functional Scope)
- 検索結果が複数ある場合、楽天 RapidAPI のデフォルトの順序をそのまま使用して先頭アイテムを選択する。
- 価格が取得できない場合、価格フィールドを null として明示的に含める。

### エッジケース
- 楽天 RapidAPI のエラー（タイムアウトや認証エラー以外）が発生した場合、HTTP 500 を返す。
- レートリミット（429 エラー）が発生した場合、楽天 RapidAPI の 429 エラーをそのまま透過する。

### 非機能要件 (Non-Functional Constraints)
- 95% のリクエストが 2 秒以内に成功応答を返す目標は、楽天 RapidAPI の応答時間を除いたシステム内部の処理時間のみを対象とする。
