# CI 負荷試験導入計画（T029）

## 目的
`SC-001` を継続的に監視するため、CI 上で `k6` による負荷試験を定期実行します。

## 方針
- PR 時は軽量シナリオ（短時間・低負荷）
- `main` マージ後または夜間に本番相当シナリオ（長時間・中負荷）
- 外部 API 依存は不安定要因になるため、CI では原則モック環境を使用

## Secrets / 設定
- `RAPIDAPI_KEY` は CI Secrets に保存
- 本番キーは CI で直接使用せず、検証用キーまたはモックを優先
- `BASE_URL` はジョブごとに環境変数で注入

## 提案ジョブ構成
1. Build & Unit Test
2. Contract Test（`acceptance-tests.md` と JSON スキーマ整合）
3. Load Test (k6)

## 失敗条件
- `p(95) >= 2000ms`
- `http_req_failed >= 5%`
- 連続 3 回で閾値超過した場合は回帰と見なす

## 運用
- 週次でトレンドを確認
- 閾値はデータを見て調整するが、仕様の `SC-001` を下回らないこと
- 負荷試験結果はアーティファクトとして保存する
