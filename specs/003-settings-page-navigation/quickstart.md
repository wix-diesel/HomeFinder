# Quickstart: 設定画面遷移導線追加

このクイックスタートは、一覧右上の歯車アイコンから設定画面へ遷移できる導線を実装・確認するための手順です。

## 1. 依存関係と起動

```bash
git checkout 003-settings-page-navigation
cd src/frontend
npm install
npm run dev
```

確認ポイント:
- 一覧画面 (`/items`) が表示される
- ヘッダー右上に歯車アイコンが表示される

## 2. 実装順序

1. ルーターへ設定画面ルート (`/settings`) を追加する。
2. 設定画面 (`SettingsPage`) を作成し、日本語文言で表示する。
3. 一覧画面ヘッダー右上へ歯車アイコンを配置する。
4. 歯車アイコンに日本語ラベルとフォーカス可視化を適用する。
5. クリック/キーボード実行で設定画面へ遷移することを確認する。

## 3. テスト実行

```bash
# frontend unit
cd src/frontend
npm run test:run

# backend contract regression (API 破壊がないことを回帰確認)
cd ../backend
dotnet test tests/contract/contract.csproj
```

## 4. 受け入れ確認

- 一覧画面右上の歯車アイコンが常時視認可能。
- クリック操作で `/settings` へ遷移する。
- キーボードでフォーカス可能、実行操作で `/settings` へ遷移する。
- フォーカス可視化が確認できる。
- 設定画面の可視テキストが日本語である。
- 設定画面から一覧へ戻る導線で `/items` へ戻れる。
- 設定項目リンクは表示のみで、遷移先を実装していない。

## 5. 成功基準の測定手順

- SC-001: 5人以上で一覧画面初期表示から10秒以内に設定導線を特定できるか確認し、成功率を記録する。
- SC-002: クリック遷移とキーボード遷移を各10回以上実施し、設定画面到達成功率を記録する。
- SC-003: キーボード操作テストの全件成功を確認する。
- SC-004: 設定画面の可視文言をレビューし、日本語表示率100%を確認する。

## 6. 完了条件

- FR-001〜FR-010 を満たす UI 挙動を確認できる。
- SC-001〜SC-004 に対応する検証観点がテストまたは手順として記録される。
- 既存 API 契約に変更がない。
