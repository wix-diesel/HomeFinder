# Research: 設定画面遷移導線追加

## Decision 1: 導線実装方式

- Decision: 一覧画面ヘッダー右上の歯車アイコンを、ルーター遷移を行うボタン/リンクとして実装する。
- Rationale: FR-001/FR-002/FR-008 の「常時視認できる 1 操作導線」を最短で満たせるため。
- Alternatives considered:
  - サイドメニュー内へ設定導線を格納: 発見性が下がり、1 操作到達要件を満たしにくいため不採用。
  - 画面下部ナビのみで提供: 一覧右上配置の要件に一致しないため不採用。

## Decision 2: ルーティング方針

- Decision: Vue Router に設定画面ルート (`/settings`) を追加し、一覧画面から名前付きルート遷移を行う。
- Rationale: 既存構成 (`/items`, `/items/new`, `/items/:id`) と整合し、テストで遷移先を安定検証できるため。
- Alternatives considered:
  - window.location による直接遷移: SPA の履歴・保守性・テスト容易性で不利なため不採用。

## Decision 3: アクセシビリティ契約

- Decision: 歯車アイコンはキーボードフォーカス可能な要素として実装し、目的が分かる日本語ラベルを必須化する。
- Rationale: FR-005/FR-006/FR-009 を確実に満たし、スクリーンリーダー利用時の意味伝達を担保できるため。
- Alternatives considered:
  - 視覚アイコンのみでラベルなし: 支援技術で目的が伝わらず要件不適合のため不採用。

## Decision 4: 設定画面のスコープ制御

- Decision: 設定画面は表示専用で実装し、項目リンクは非活性またはダミー表示にとどめる。
- Rationale: FR-007 の「リンク先遷移は本スコープ外」を守りつつ、UI 構成再現要件を満たせるため。
- Alternatives considered:
  - 各設定項目の遷移先を同時実装: 仕様範囲外で実装規模が拡大するため不採用。

## Decision 5: 日本語表示統一

- Decision: 設定画面の見出し・説明・操作ラベルを日本語定数として管理し、可視テキストの英語混在を禁止する。
- Rationale: FR-003 と SC-004 (日本語表示率 100%) をレビュー可能な形で保証しやすいため。
- Alternatives considered:
  - 文言をコンポーネントへ直接ハードコード: 表記ゆれ・レビュー漏れのリスクが高く不採用。

## Clarification Resolution

- 一覧右上配置、クリック/キーボード実行対応、参照デザイン範囲 (`design/items_list.html`, `design/settings.html`) の前提を設計へ反映した。
- 設定画面内リンクは本機能では遷移実装しない方針を確定した。
- Technical Context の NEEDS CLARIFICATION は 0 件。

## Measurement Protocol

### SC-001: 導線認識率

- 対象: 5人以上の確認者
- 条件: 一覧画面の初期表示状態から開始し、事前に導線位置は説明しない
- 手順: 10秒以内に「設定へ行くにはどこを操作するか」を指差しまたは口頭で回答してもらう
- 記録: 成功/失敗、所要秒数、見落とし要因メモを記録する
- 判定: 成功者数 / 参加者数 が 95% 以上で合格

### SC-002: 遷移成功率

- 対象: クリック遷移 10回以上、キーボード遷移 10回以上
- 条件: 一覧画面から開始し、設定画面がエラー通知なしで描画完了した場合を成功とする
- 手順: クリック操作と Enter/Space 操作をそれぞれ反復し、成功/失敗を記録する
- 記録: 操作種別、試行回数、成功回数、失敗時の症状を記録する
- 判定: 成功回数 / 総試行回数 が 99% 以上で合格
