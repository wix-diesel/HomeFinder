# Feature Specification: 設定画面遷移導線

**Feature Branch**: `003-settings-page-navigation`  
**Created**: 2026-04-24  
**Status**: Draft  
**Input**: User description: "設定画面作成と、一覧右上の歯車アイコンから設定画面へ遷移できる導線を追加する"

## 目的（Why）

- アイテム一覧から設定画面へ迷わず移動できる導線を提供し、操作の発見性を高める。
- designフォルダのHTML（`design/items_list.html` と `design/settings.html`）を参照して設定画面を提供し、画面間の一貫した体験を実現する。

## Clarifications

### Session 2026-04-24

- 歯車アイコンの配置位置は、一覧画面ヘッダー領域の右上とする。
- 設定画面への遷移操作は、クリックに加えてキーボードの実行操作でも成立させる。
- 参照デザインの対象は designフォルダの `design/items_list.html` と `design/settings.html` に限定する。
- 設定画面内の各項目リンクは、本機能では遷移先を実装しない（表示のみ）。

## User Scenarios & Testing *(mandatory)*

## ユーザーストーリー

### User Story 1 - 一覧から設定画面へ移動する (Priority: P1)

利用者として、アイテム一覧ページの右上にある歯車アイコンから設定画面へ遷移したい。これにより、必要な設定操作へ最短で到達できる。

**Why this priority**: 設定画面に到達できないと本機能の価値が成立しないため、最優先とする。

**Independent Test**: 一覧ページから歯車アイコンを1回操作し、設定画面が表示されることを単体で確認できる。

**受け入れ条件（AC）**:

1. **Given** 利用者がアイテム一覧ページを表示している, **When** 右上の歯車アイコンをクリックする, **Then** 設定画面へ遷移する。
2. **Given** 利用者がキーボード操作のみで一覧ページを操作している, **When** 歯車アイコンにフォーカスして実行操作する, **Then** 設定画面へ遷移する。

---

### User Story 2 - 設定画面を日本語で理解する (Priority: P2)

利用者として、設定画面の表示内容を日本語で理解したい。これにより、迷わず設定項目を把握できる。

**Why this priority**: 遷移後の画面理解ができないと実利用時の操作性が下がるため。

**Independent Test**: 設定画面単体を表示し、画面内の可視テキストが日本語であることを確認できる。

**受け入れ条件（AC）**:

1. **Given** 利用者が設定画面を表示している, **When** 画面内の見出し・説明・ボタンを確認する, **Then** すべて日本語で表示される。
2. **Given** 利用者が一覧画面と設定画面を見比べる, **When** レイアウトと情報配置を確認する, **Then** `design/items_list.html` と `design/settings.html` に沿った構成で表示される。
3. **Given** 利用者が設定画面を表示している, **When** 一覧へ戻る導線を操作する, **Then** アイテム一覧画面へ戻れる。

---

### User Story 3 - アクセシブルに導線を使う (Priority: P3)

キーボード利用者として、歯車アイコンを視認しながら確実に操作したい。これにより、マウスに依存せず設定画面へ到達できる。

**Why this priority**: 主要導線の操作可能性を確保することで、利用者の取りこぼしを防ぐため。

**Independent Test**: 一覧ページでキーボードのみを使って歯車アイコンへ到達し、フォーカス可視化と遷移成立を確認できる。

**受け入れ条件（AC）**:

1. **Given** 一覧画面が表示されている, **When** 利用者がキーボードでフォーカス移動する, **Then** 歯車アイコンにフォーカス可能で、視認できるフォーカス表示が出る。
2. **Given** 利用者が支援技術で操作している, **When** 歯車アイコンのラベルを読む, **Then** アイコンの目的が分かる名称で案内される。

---

### Edge Cases

## エッジケース/失敗時の挙動

- 歯車アイコンの表示領域が狭い場合でも、誤タップしにくい操作領域を維持する。
- 設定画面への遷移先が一時的に表示できない場合は、利用者に失敗を伝え、一覧画面で再試行可能な状態を維持する。
- 画像やアイコン素材が読み込めない場合でも、設定遷移操作の意味が分かる代替表示を維持する。
- キーボード操作時にフォーカスが見えない状態を作らず、現在位置を常に判別可能にする。

## Requirements *(mandatory)*

## 要求（Must / Should / Could）

### Must

- アイテム一覧ページ右上に、設定画面への遷移を示す歯車アイコンを表示しなければならない。
- 利用者が歯車アイコンを操作したとき、設定画面へ遷移しなければならない。
- 設定画面の可視テキストは日本語で表示しなければならない。
- 一覧画面と設定画面の見た目は、designフォルダのHTML（`design/items_list.html` と `design/settings.html`）の構成を再現しなければならない。
- 歯車アイコンはキーボード操作とフォーカス可視化に対応し、目的が分かるラベルを提供しなければならない。

### Should

- 利用者は一覧画面表示中に、1操作で設定画面へ到達できるべきである。
- 設定画面は一覧画面から遷移した利用者が迷わないよう、視覚的一貫性を保つべきである。

### Could

- 歯車アイコンにホバー時の補助表現を追加し、設定導線の発見性を高めてもよい。

### Functional Requirements

- **FR-001**: システムは、アイテム一覧ページの右上に設定画面遷移用の歯車アイコンを表示しなければならない。
- **FR-002**: システムは、歯車アイコンの操作で設定画面へ遷移させなければならない。
- **FR-003**: システムは、設定画面内の可視テキストを日本語で提供しなければならない。
- **FR-004**: システムは、一覧画面を `design/items_list.html`、設定画面を `design/settings.html` の情報配置・セクション構成および視覚スタイル（色・タイポグラフィ・レイアウト）を再現して表示しなければならない。ただし既存の CSS や TypeScript で同等の構成が実装済みの場合はそれを適宜再利用してよい。
- **FR-005**: システムは、歯車アイコンをキーボードで操作可能にし、フォーカス状態を視覚的に示さなければならない。
- **FR-006**: システムは、歯車アイコンに対して利用目的が分かるラベルを提供しなければならない。
- **FR-007**: システムは、設定画面に表示される各項目へのリンク機能を本機能の範囲に含めてはならない。
- **FR-008**: システムは、一覧画面表示中に歯車アイコンが常に視認できる状態を維持しなければならない。
- **FR-009**: システムは、歯車アイコンのキーボード実行操作後に設定画面へ遷移しなければならない。
- **FR-010**: システムは、設定画面内にアイテム一覧へ戻るための導線を1つ以上提供しなければならない。

### Key Entities *(include if feature involves data)*

- **設定導線アイコン**: 一覧画面上で設定画面への移動を開始する操作要素。
- **設定画面表示要素**: 設定画面で利用者に見える見出し、説明、ボタンなどの表示単位。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 一覧画面を表示した利用者の95%以上が、10秒以内に設定画面への遷移導線（歯車アイコン）を認識できる。
- **SC-002**: 一覧画面から設定画面への遷移成功率が99%以上である。
- **SC-003**: キーボード操作のみで設定画面へ到達できるテスト成功率が100%である。
- **SC-004**: 設定画面の可視UI文言の日本語表示率が100%である。

### Success Criteria Measurement Notes

- **SC-001 測定方法**: 5人以上の確認者が一覧画面初期表示から10秒以内に設定導線を特定できたかを観察し、成功者数 / 参加者数で算出する。合格条件は成功率95%以上とする。
- **SC-002 測定方法**: クリック遷移とキーボード遷移を各10回以上実施し、設定画面へ到達してエラー通知なしで表示完了した件数 / 総試行回数で算出する。合格条件は成功率99%以上とする。

## Assumptions

- 対象は既存のアイテム一覧画面利用者であり、同一アプリ内で画面遷移する。
- 設定画面の各項目詳細へのリンク先定義は本スコープ外とする。
- designフォルダにある `design/items_list.html` と `design/settings.html` は、一覧画面と設定画面の情報構成の基準として確定している。
- 歯車アイコンのクリック・キーボード実行後の UI 応答は、通常利用環境で 100ms 以内を目標とする。これは計測・検証タスクの対象外であり、実装時の設計指針として適用する。

## 用語

- **一覧画面**: アイテムの一覧を表示する画面。
- **設定画面**: アプリ設定に関する表示項目を確認する画面。
- **歯車アイコン**: 一覧画面から設定画面へ移動するための操作要素。
