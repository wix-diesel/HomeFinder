# Quickstart: アイテム画面デザイン再構成

このクイックスタートは、一覧画面/登録画面の再デザイン実装を開始し、共通コンポーネント化と日本語 UI を検証するための手順です。

## 1. ブランチと依存関係

```bash
git checkout 002-before-specify-hook
cd src/frontend
npm install
```

## 2. 開発サーバー起動

```bash
npm run dev
```

確認ポイント:
- 一覧画面が表示される
- 登録画面へ遷移できる
- UI 文言が日本語で表示される
- 一覧画面に検索、カテゴリ、表示切替、登録開始導線が表示される

## 3. 実装順序

1. 共通 UI 基盤を作成する (`AppLayout`, ヘッダー, ボタン, 入力, 状態表示)。
2. 一覧画面を再構成する (検索、カテゴリチップ、カード、デスクトップ切替)。
3. 登録画面を再構成する (日本語ラベル、ヘルプ、必須検証、送信中制御)。
4. payload マッピングを実装し、UI-only 項目を API 送信から除外する。
5. 登録成功時に一覧へ遷移し、成功トーストを表示する。

## 4. テスト実行

```bash
# frontend unit
cd src/frontend
npm run test:run

# backend contract regression
cd ../backend
dotnet test tests/contract/contract.csproj
```

補足:
- 契約テストが環境依存で長時間化する場合、`--logger "console;verbosity=minimal"` を付与して実行ログを簡略化する。

## 5. 受け入れ確認

- 一覧画面:
  - キーワード検索が即時反映される
  - カテゴリ絞り込みが機能する
  - モバイルはカード固定、デスクトップはカード/テーブル切替可能
- 登録画面:
  - 必須不足時に日本語エラー表示
  - 送信中は主要アクションが無効化
  - 失敗時に入力保持 + 再試行可能
  - 成功時に一覧遷移 + 成功トースト
- 共通化:
  - 空状態/検証エラー/送信中/失敗が共通コンポーネントで表示される
  - 共通見出しと表示切替が一覧/登録で再利用される

## 6. 完了条件

- FR-001 から FR-010 を満たす UI 挙動が確認できる。
- 日本語表示率 100% (ラベル、ボタン、エラー、ヘルプ、空状態) を満たす。
- 既存 Items API 契約を変更せず、登録 payload に UI-only 項目が含まれない。
