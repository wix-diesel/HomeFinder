# 機能仕様: ユーザー設定画面

**フィーチャーブランチ**: `012-user-settings`  
**作成日**: 2026-05-10  
**ステータス**: ドラフト  

## Clarifications

### Session 2026-05-10

- Q: アイコン画像の対応形式とサイズ制限は？ → A: PNG・JPG のみ対応、最大 2MB
- Q: 初回ログイン時の「未設定用アイコン」の具体的な定義は？ → A: デフォルトユーザーアイコン（グレーの人型シルエット）をシステムで定義して使用
- Q: 保存失敗時のエラーハンドリング戦略は？ → A: エラー時はトースター表示し、入力エラーはフィールド単位で表示して再試行可能にする
- Q: プロフィール更新の権限境界は？ → A: ログインユーザーは自分自身のプロフィールのみ更新可能
- Q: 表示名の入力制約は？ → A: 表示名は 1〜30 文字、任意文字（絵文字含む）

## ユーザーシナリオとテスト *(必須)*

### ユーザーストーリー 1 - プロフィール情報の表示と編集 (優先度: P1)

ユーザーがアプリを利用する中で、自分の登録名称やアイコンを確認・変更したいと考えています。ユーザーアイコンをクリックしてプロフィール画面に遷移し、名前とアイコンの確認・編集が可能な画面が提供されます。

**この優先度の理由**: ユーザーのプロフィール管理は基本的な機能であり、初回ログイン後すぐに必要となる機能です。

**独立テスト**: ユーザーが右上のアイコンをクリック → プロフィール画面が表示される → 名前・アイコンを変更 → 保存ボタンで変更を保存 → トースター表示で完全にテストでき、ユーザーにプロフィール管理機能を提供します。

**受け入れシナリオ**:

1. **Given** ユーザーがログイン後のメイン画面にいる, **When** 右上のユーザーアイコンをクリック, **Then** ユーザー設定画面に遷移する
2. **Given** ユーザーが設定画面にいる, **When** 画面を表示する, **Then** ユーザーアイコン・名前・メールアドレスが表示される
3. **Given** ユーザーが設定画面にいる, **When** 名前とアイコンを変更して保存ボタンを押す, **Then** 保存完了のトースターが表示される
4. **Given** ユーザーが設定画面で変更を保存した, **When** 画面を遷移して戻る, **Then** 変更が反映されている

---

### ユーザーストーリー 2 - 初回ログイン時のプロフィール初期化 (優先度: P2)

ユーザーが初めてシステムにログインした際、ユーザー名とアイコンが自動的に初期化されます。これにより、全てのユーザーが一貫した初期状態からスタートし、その後必要に応じてカスタマイズできます。

**この優先度の理由**: 初回ログイン体験は重要ですが、P1 のプロフィール管理機能の実装後に対応可能です。

**独立テスト**: 初回ログインユーザーがプロフィール画面を確認 → 名前がメールアドレスで初期化、アイコンが未設定画像になっている → ユーザーが意図した値に変更可能、で完全にテストできます。

**受け入れシナリオ**:

1. **Given** ユーザーが初回ログインした, **When** プロフィール画面を表示する, **Then** 名前はメールアドレスの値で初期化されている
2. **Given** ユーザーが初回ログインした, **When** プロフィール画面を表示する, **Then** アイコンは未設定用の画像が表示されている
	- 未設定用の画像は、グレーの人型シルエットのデフォルトアイコンとする

---

### ユーザーストーリー 3 - 他の画面でのプロフィール情報表示 (優先度: P3)

ユーザーが設定画面など、プロフィール以外の画面でも変更したプロフィール情報（名前・アイコン）が反映されます。これにより、ユーザーは全アプリにおいて一貫したプロフィール情報を見ることができます。

**この優先度の理由**: UI の一貫性を保つための機能ですが、ユーザーの基本的なニーズ（プロフィール変更）の実装後に対応可能です。

**独立テスト**: ユーザーがプロフィール画面で名前・アイコンを変更 → 設定画面など他の画面を確認 → 変更が反映されている、で完全にテストできます。

**受け入れシナリオ**:

1. **Given** ユーザーがプロフィール画面で名前を変更して保存した, **When** 設定画面に遷移する, **Then** 設定画面に新しい名前が表示されている
2. **Given** ユーザーがプロフィール画面でアイコンを変更して保存した, **When** アプリの任意の画面を確認する, **Then** 新しいアイコンが表示されている

---

### エッジケース

- **ネットワークエラー**: 保存中にネットワーク接続が失われた場合、エラーメッセージを表示し、ユーザーに再試行を促す
- **同時アクセス**: 複数のタブ/デバイスから同時にプロフィール情報を編集した場合、最後に保存された値が有効になる
- **アイコンファイル形式制限**: ユーザーがアップロードするアイコン画像は PNG または JPG 形式に限定し、最大 2MB まで対応。2MB を超える場合または形式が異なる場合は、ユーザーに形式変更を促すエラーメッセージを表示する。自動リサイズにより最終画像は 500×500px に統一される
- **デフォルトアイコン**: 初回ログイン時に、ユーザーのアイコンはシステムで定義したデフォルトアイコン（グレーの人型シルエット）で表示される。ユーザーが任意のタイミングで変更可能
- **メールアドレスの変更不可**: メールアドレスフィールドは読み取り専用であり、ユーザーが編集できないことが明確に表示される
- **権限不足**: 自分以外のユーザープロフィール更新リクエストは認可エラーとして拒否し、失敗トースターを表示する
- **表示名の入力制約**: 表示名が 0 文字または 31 文字以上の場合は保存不可とし、名前フィールドにバリデーションエラーを表示する

## 要件 *(必須)*

### 機能要件

- **FR-001**: ユーザーはアプリのヘッダー（右上）に表示されたユーザーアイコン、または設定画面のプロフィール領域をクリックしてユーザー設定画面にアクセスできること
- **FR-002**: ユーザー設定画面にはユーザーのアイコン、名前、メールアドレスが表示されること
- **FR-003**: ユーザーはユーザー設定画面でアイコンと名前を編集できること
- **FR-004**: ユーザーはメールアドレスを編集できないこと（読み取り専用表示）
- **FR-005**: ユーザーがアップロードするアイコンは PNG または JPG 形式のみ対応し、最大ファイルサイズは 2MB とすること
- **FR-006**: ユーザーが保存ボタンを押すと、変更内容がシステムに保存されること
- **FR-007**: 保存が成功した場合、成功トースターを表示し、ページ遷移は行わないこと
- **FR-008**: 保存が失敗した場合、失敗理由をトースターで表示し、ユーザーが再試行できること
- **FR-009**: 入力値に不備がある場合、該当フィールドごとにバリデーションエラーを表示すること
- **FR-010**: ユーザーはページ遷移なしに設定画面を利用し続けられること
- **FR-011**: 初回ログイン時に、ユーザーの名前はメールアドレスで初期化されること
- **FR-012**: 初回ログイン時に、ユーザーのアイコンはデフォルトユーザーアイコン（グレーの人型シルエット）で初期化されること
- **FR-013**: 設定画面など他の画面でも、ユーザーが変更したプロフィール情報（名前・アイコン）が反映されること
- **FR-014**: ユーザー設定画面のデザインは `/design/user_settings.html` に準拠すること
- **FR-015**: ログインユーザーは自分自身のプロフィールのみ更新可能であり、他ユーザー情報の更新はできないこと
- **FR-016**: 表示名は 1〜30 文字で入力でき、任意文字（絵文字含む）を許可すること

### 主要エンティティ

- **User**: ユーザーの基本情報を表すエンティティ。主要属性としてユーザー ID、メールアドレス、表示名、アイコンを持つ

## 成功基準 *(必須)*

### 測定可能な成果

- **SC-001**: ユーザーが 3 ステップ以内（画面表示 → 情報編集 → 保存）でプロフィール変更を完了できる
- **SC-002**: プロフィール画面の表示から保存完了まで 2 秒以内に完了する
- **SC-003**: 保存後、変更内容が即座に反映される（1 秒以内）
- **SC-004**: 95% 以上のユーザーが初回試行でプロフィール変更に成功する
- **SC-005**: ユーザー設定画面が `/design/user_settings.html` のデザイン仕様に 100% 準拠する

## 前提条件

- ユーザーは MS Entra ID で認証されている
- ユーザーは安定したインターネット接続を持っている
- ユーザープロファイル API は未実装であり、本フィーチャーで新規実装する
- アイコンアップロード機能は既に実装されている（あるいは別フィーチャーで実装予定）
- トースター通知コンポーネントは既に実装されている
- ページ遷移なしに API 呼び出しを実行できるフロントエンド環境が整備されている
